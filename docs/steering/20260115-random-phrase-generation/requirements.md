# ランダムフレーズ生成 - 要件定義

**ステータス**: 要件確定
**作成日**: 2026-01-15
**最終更新日**: 2026-01-15

---

## 1. 背景と課題

### 解決したい課題

ユーザーがフレーズを追加したいとき、翻訳するネタ（フレーズ）を自分で考える必要がある。しかし、何を翻訳すればいいか思いつかないことがあり、学習の障壁になっている。

### 現状の問題点

- 既存のフレーズ生成機能では、毎回似たような表現が生成されがち
- ユーザーが学習したい表現のバリエーションが限られる

---

## 2. 期待する結果

### ユーザーストーリー

- ユーザーとして、翻訳したいフレーズを考えなくても、ボタン1つでフレーズ候補を得られるようになりたい。なぜなら、ネタ切れで学習が止まることを防ぎたいから
- ユーザーとして、毎回異なる構文パターンのフレーズが生成されるようになりたい。なぜなら、バリエーション豊かな学習がしたいから

### ビジネスインパクト

- フレーズ追加のハードルが下がり、学習継続率が向上する
- より多様なフレーズを学習することで、学習効果が高まる

---

## 3. ゴール

### 実現すること（ゴール）

- [x] トグルで通常のフレーズ生成とランダム生成を切り替えられる
- [x] ランダム生成ボタンを押すと、97個の構文パターンからランダムに選択してフレーズを生成
- [x] フレーズに「本文（学習中の言語）」「翻訳」「構文パターンの解説」を表示
- [x] ユーザーが生成されたフレーズを選択して保存できる
- [x] 使用回数は通常のフレーズ生成と合計で5回まで

### 今回は対象外（非ゴール）

- 構文パターンのカスタマイズ機能
- 生成フレーズの編集機能（選んだものをそのまま保存）

## 5. 制約条件

### 技術的制約

- 既存のフレーズ生成機能のUI・ロジックとの整合性を保つ
- 使用回数制限は既存のフレーズ生成機能と共有（合計5回）

### ビジネス制約

- AI APIの呼び出しコストを考慮

### 前提条件

- ユーザーはログイン済み
- ユーザーは学習言語を設定済み
